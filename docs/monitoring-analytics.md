# Monitoring and Analytics

Phase 4.5 records only existing authenticated `RouterSnapshot` evidence. Huawei collection remains in the driver and existing non-overlapping polling service. WAN byte counters are stored raw; rates are derived as `byte delta × 8 / elapsed seconds`. Reset, missing, duplicate, and out-of-order samples remain gaps.

Metrics carry value, unit, source, classification, collection time, age, stale state, confidence, and support state. WebSocket `monitoring.snapshot` events are versioned and originate only from real polling snapshots. Per-device and per-interface traffic are unsupported until verified counters exist.

SQLite tables cover metric and counter samples, traffic, availability, public-IP history, events, alert rules and lifecycle, collection diagnostics, and retention runs. Indexed queries are capped at 2,000 points.
